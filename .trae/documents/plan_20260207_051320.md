I have added the **Upload Purchase Order** feature. This allows you to upload the original PO file (PDF/Image) directly to the Purchase Order record in the system for easy reference.

## 1. Database Schema

### Updated Table: `purchase_orders`
*   Added field: `file_attachment` (VARCHAR, nullable) - Stores the path to the uploaded file.

### Complete Schema Overview
*   **Customers**: `name`, `company_name`, `address`, `vat_number`, `created_by`.
*   **Quotations**: `quotation_number`, `items`, `totals`, `created_by`.
*   **Invoices**: `invoice_number`, `items`, `payment_status`, `created_by`.
*   **Purchase Orders**:
    *   **Fields**: `po_number`, `buyer_id`, `po_date`, `purchaser_name`, `payment_term`, `currency`.
    *   **New Field**: `file_attachment` (File Upload).
    *   **Items**: `item_sequence`, `material`, `desc`, `qty`, `uom`, `delivery_date`, `price`.

## 2. Filament Components

### PurchaseOrderResource
*   **Form**:
    *   **Header**: Existing fields (PO No, Buyer, Date, etc.).
    *   **New Field**: `FileUpload::make('file_attachment')`
        *   Allowed types: PDF, Images.
        *   Directory: `purchase-orders`.
        *   Visibility: Private (or Public, based on security needs).
        *   **Purpose**: "Supaya bisa langsung tau reference nya" - Users can click to view the original PO document.
    *   **Items**: Repeater.

## 3. Implementation Steps
1.  **Migrations**: Run SQL schema (including `file_attachment` column).
2.  **Filament**:
    *   Install Shield.

    *   Add `FileUpload` component to `PurchaseOrderResource`.
    *   Implement PDF Export for Quotations.
